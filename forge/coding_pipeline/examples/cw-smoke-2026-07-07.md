## The setup

Three leaves emitted to Forge project `Pipeline-Smoke` under epic slug `cw-smoke`, all
Auto-OK / `requires_tests=False` / `max_files=1`, **deliberately without `file_scope`** so
the batch picker disengages and the colliding pair actually reaches the reconcile barrier:

- **Smoke A** — replace README.md's first line with `# unitconv — tiny unit conversion CLI (variant alpha)`
- **Smoke B** — replace the SAME line with `# unitconv — the smallest unit converter (variant beta)` (guaranteed textual conflict with A)
- **Smoke C** — create `docs/USAGE.md` (disjoint)

Invocation (from the toy repo's own cwd — `meta build run` takes its repo from `Path.cwd()`;
never wrap it in `uv run --directory <elsewhere>`, which changes the cwd):

```
cd ~/Projects/erewhon/pipeline-smoke
meta build run cw-smoke --project Pipeline-Smoke --concurrency 3
```

## Infra findings fixed en route (own commits, live-verified)

1. gaol run-once scheduled containers onto a remote incus **cluster** member — mounts can
   never exist there (gaol `2806736`: pin `--target` to the local member).
2. The NAT'd sandbox can't resolve LAN/mesh DNS names and DHCP needs ~5–10s (gaol
   `0676c60`: `--add-host`, `--wait-network`; task_worker sandbox rides both).
3. Out-of-repo path deps are invisible inside workspace sandboxes (meta's
   `../nous/nous-py`) — `runonce_extra_mounts` (+ shared uv cache) fixed it; proven by a
   full task_worker suite run inside a run-once sandbox on a fresh meta workspace.

## Reproduce

Re-emit three leaves like the above (any two rewriting one shared line, one disjoint;
no `file_scope` on any of them) to a toy repo with a dx container, then run the wave loop
at `--concurrency 3` from that repo's cwd. The expected outcome is this document's wave-4
shape: one collider lands, the other is demoted with a conflict note, the disjoint leaf
lands, and the demoted leaf lands cleanly on the next wave after re-arming.
